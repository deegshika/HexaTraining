# Cloud Computing Basics

## What is Cloud Computing?

Cloud Computing is the delivery of computing services such as servers, storage, databases, networking, software, and analytics over the internet instead of using a local computer or physical server.

In simple words, cloud computing means storing and accessing data or applications through the internet rather than on your own device.

### Example:

* Storing photos in Google Drive
* Watching movies on Netflix
* Using Gmail or Google Docs online

Instead of buying expensive hardware, companies can use cloud services and pay only for what they use.

### Key Features of Cloud Computing

1. **On-demand availability** – Services are available anytime.
2. **Scalability** – Resources can increase or decrease based on needs.
3. **Cost-effective** – No need to buy expensive servers.
4. **Accessibility** – Access from anywhere with internet.
5. **Security** – Cloud providers offer data protection and backups.

### Types of Cloud

#### 1. Public Cloud

Services are available to everyone over the internet.

**Example:** Microsoft Azure, AWS, Google Cloud

#### 2. Private Cloud

Cloud infrastructure used only by one organization.

**Example:** Banking company private servers

#### 3. Hybrid Cloud

Combination of public and private cloud.

**Example:** Companies storing sensitive data privately and public apps on cloud.

# Advantages of Microsoft Azure

Microsoft Azure is a cloud computing platform provided by Microsoft for building, hosting, and managing applications.

### Advantages of Azure

### 1. High Security

Azure provides strong security, encryption, and identity management to protect data.

### 2. Scalability

Businesses can increase or decrease resources depending on workload.

### 3. Cost Effective

Pay-as-you-go model helps companies pay only for services used.

### 4. High Availability

Azure services are available almost all the time with minimum downtime.

### 5. Easy Integration

Azure works well with Microsoft tools like:

* Windows Server
* Microsoft Office
* SQL Server
* Visual Studio

### 6. Global Reach

Azure has data centers across many countries, allowing faster service worldwide.

### 7. Backup and Recovery

Azure helps in automatic backup and disaster recovery.

### Real-Life Example

If an e-commerce website gets more users during a sale, Azure automatically increases server resources to handle traffic.

# What is Hosting in Azure?

Hosting in Azure means deploying and running websites, applications, databases, or services on Microsoft Azure servers instead of local systems.

Azure provides infrastructure to host:

* Websites
* Mobile apps
* APIs
* Databases
* Virtual machines

### Example

If you create a React website or Python web app, instead of running it only on your laptop, you can host it on Azure so people around the world can access it using the internet.

### Common Azure Hosting Services

#### 1. Azure App Service

Used for hosting websites and web applications.

#### 2. Azure Virtual Machine (VM)

Used for creating virtual computers on cloud.

#### 3. Azure Blob Storage

Used for storing files, videos, and documents.

#### 4. Azure SQL Database

Used for storing application databases.

### Benefits of Azure Hosting

* Fast performance
* Global access
* Automatic scaling
* Better security
* Reduced maintenance cost

# What are IaaS, PaaS and SaaS?

Cloud services are mainly divided into three models:

## 1. IaaS (Infrastructure as a Service)

IaaS provides virtualized hardware resources over the internet.

The cloud provider manages:

* Networking
* Storage
* Physical servers

The user manages:

* Applications
* Operating systems
* Data

### Example:

* Azure Virtual Machines
* Amazon EC2

### Use Case:

If a company wants full control over software and operating systems.

### Advantages:

* High flexibility
* Full control
* Easy scalability

---

## 2. PaaS (Platform as a Service)

PaaS provides a platform to develop, test, and deploy applications without managing hardware.

The cloud provider manages:

* Hardware
* Operating system
* Runtime environment

The user manages:

* Applications
* Data

### Example:

* Azure App Service
* Google App Engine

### Use Case:

Developers who want to focus only on coding.

### Advantages:

* Faster development
* Less maintenance
* Easy deployment

---

## 3. SaaS (Software as a Service)

SaaS provides software applications through the internet.

The cloud provider manages everything.

Users only access and use the software.

### Example:

* Gmail
* Google Docs
* Microsoft Office 365
* Zoom

### Use Case:

Users who only need software without installation.

### Advantages:

* No installation needed
* Accessible anywhere
* Automatic updates

---

# Difference Between IaaS, PaaS and SaaS

| Feature             | IaaS           | PaaS                | SaaS       |
| ------------------- | -------------- | ------------------- | ---------- |
| Control Level       | High           | Medium              | Low        |
| Managed By Provider | Hardware       | Platform + Hardware | Everything |
| User Manages        | OS, Apps, Data | Apps, Data          | Nothing    |
| Example             | Azure VM       | Azure App Service   | Gmail      |

### Simple Memory Trick

* **IaaS → Infrastructure** (Control more things)
* **PaaS → Platform** (Focus on coding)
* **SaaS → Software** (Just use software)

## Conclusion

Cloud Computing makes technology more flexible, scalable, and cost-effective. Microsoft Azure helps businesses host applications securely and efficiently. Azure hosting allows applications to be accessed globally, while IaaS, PaaS, and SaaS explain different ways cloud services are delivered based on user needs.
