# DevOps and Agile Notes

# What is DevOps?

DevOps is a software development approach that combines **Development (Dev)** and **Operations (Ops)** to improve collaboration, automate processes, and deliver software faster with better quality.

### Goals of DevOps

* Faster software delivery
* Better collaboration between teams
* Continuous Integration and Continuous Deployment (CI/CD)
* Improved software quality
* Faster issue detection and recovery

### DevOps Lifecycle

1. Plan
2. Develop
3. Build
4. Test
5. Release
6. Deploy
7. Operate
8. Monitor

# What is Agile Methodology?

Agile is a software development methodology that delivers software in **small, incremental releases** instead of one large release.

It focuses on:

* Customer collaboration
* Continuous improvement
* Frequent delivery
* Flexibility to changing requirements

### Agile Principles

* Deliver working software frequently.
* Welcome changing requirements.
* Maintain close collaboration with customers.
* Encourage teamwork.
* Continuously improve the development process.

---

# What is Scrum?

Scrum is the most popular Agile framework used to manage software development projects.

Development work is divided into short iterations called **Sprints**, usually lasting **2–4 weeks**.

### Scrum Roles

* **Product Owner** – Defines product requirements and manages the product backlog.
* **Scrum Master** – Ensures Scrum practices are followed and removes obstacles.
* **Development Team** – Designs, develops, tests, and delivers the product.

### Scrum Artifacts

* Product Backlog
* Sprint Backlog
* Increment

---

# Important Meetings in Agile (Scrum Ceremonies)

## 1. Sprint Planning

* Conducted at the beginning of every sprint.
* Team selects work from the Product Backlog.
* Sprint Goal is defined.

## 2. Daily Stand-up (Daily Scrum)

* Conducted every day for about **15 minutes**.
* Team members answer:

  * What did I complete yesterday?
  * What will I do today?
  * Are there any blockers?

## 3. Sprint Review

* Conducted at the end of the sprint.
* Team demonstrates completed work to stakeholders.
* Feedback is collected.

## 4. Sprint Retrospective

* Conducted after the Sprint Review.
* Team discusses:

  * What went well?
  * What can be improved?
  * Action items for the next sprint.

# DevOps vs Agile (Quick Comparison)

| Agile                                           | DevOps                                                          |
| ----------------------------------------------- | --------------------------------------------------------------- |
| Focuses on software development process         | Focuses on development and operations together                  |
| Uses short iterations (Sprints)                 | Uses Continuous Integration and Continuous Deployment (CI/CD)   |
| Delivers software frequently                    | Automates build, test, deployment, and monitoring               |
| Improves collaboration within development teams | Improves collaboration between development and operations teams |

# Frequently Asked Interview Questions

### 1. What is DevOps?

DevOps is a culture and set of practices that combines Development and Operations to deliver software faster through automation and collaboration.

### 2. What is Agile?

Agile is a software development methodology that delivers software in small, iterative releases with continuous customer feedback.

### 3. What is Scrum?

Scrum is an Agile framework where work is completed in fixed-duration sprints.

### 4. Name the Scrum roles.

* Product Owner
* Scrum Master
* Development Team

### 5. Name the four Scrum meetings.

* Sprint Planning
* Daily Stand-up
* Sprint Review
* Sprint Retrospective

### 6. What is a Sprint?

A Sprint is a fixed time period (usually 2–4 weeks) during which a specific set of work is completed.

### 7. What is CI/CD?

* **CI (Continuous Integration):** Frequently integrating code into a shared repository with automated testing.
* **CD (Continuous Deployment/Delivery):** Automatically delivering tested code to production or staging environments.

* Faster releases
* Better collaboration
* Improved software quality
* Faster issue resolution

### 9. Mention any four advantages of Agile.

* Faster delivery
